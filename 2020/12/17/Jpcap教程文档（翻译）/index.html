什么是Jpcap

Jpcap是一个开源库，用于从Java应用程序捕获和发送网络包。它提供以下工具：

  • 从线路上实时捕获原始数据包
  • 将捕获的数据包保存到脱机文件中，并从脱机文件中读取捕获的数据包
  • 自动识别数据包类型并生成相应的Java对象（用于以太网、IPv4、IPv6、ARP/RARP、TCP、UDP和ICMPv4数据包）
  • 在将数据包发送到应用程序之前，请根据用户指定的规则过滤这些数据包
  • 向网络发送原始数据包

Jpcap基于libpcap/winpcap，用C语言和Java语言实现

Jpcap已经在Microsoft Windows (98/2000/XP/Vista), Linux (Fedora, Ubuntu), Mac OS X (Darwin), FreeBSD和Solaris上进行了测试。

使用Jpcap可以开发什么样的应用程序

Jpcap可用于开发多种网络应用，包括（但不限于）：

  • 网络和协议分析器
  • 网络监视器
  • 交换记录（traffic loggers）
  • 交换发生器（traffic generators）
  • 用户级网桥和路由器
  • 网络入侵检测系统（NIDS）
  • 网络扫描仪
  • 安全工具

Jpcap不能做什么

Jpcap独立于主机协议（如TCP/IP）捕获和发送数据包。这意味着Jpcap不会（不能）阻止、过滤或操纵同一台机器上其他程序生成的流量：它只是“嗅探”在网上传输的数据包。因此，它不能为流量整形器、QoS调度程序和个人防火墙等应用程序提供适当的支持。

当调用JpcapCaptor.openDevice()方法，可以指定以下参数：

名称 作用
NetworkInterface intrface 要打开的网络接口
int snaplen 一次捕获的最大字节数
boolean promics 如果要以混杂模式打开接口，则为True，否则为false
在混杂模式下，您可以从线路上捕获每个数据包，即使其源或目标MAC地址与您正在打开的接口的MAC地址不相同
在非混杂模式下，只能捕获主机发送和接收的数据包
int to_ms 设置捕获超时值以毫秒为单位

JpcapCaptor.openDevice())返回JpcapCaptor的实例。然后，您可以调用JpcapCaptor类的几个方法来实际从网络接口捕获数据包。

从网络接口捕获数据包

一旦获得了JpcapCaptor的实例，就可以从接口捕获数据包。

使用JpcapCaptor实例捕获数据包有两种主要方法：使用回调方法和逐个捕获数据包。

使用回调方法

在这种方法中，实现一个回调方法来处理捕获的数据包，然后将回调方法传递给Jpcap，以便Jpcap在每次捕获数据包时都回调它。让我们看看如何详细地执行此方法。

下面的类实现了一个receivePacket()方法，该方法只输出捕获的数据包。

那么，你可以调用任何一个JpcapCaptor.processPacket()或JpcapCaptor.loopPacket()方法开始使用回调方法捕获。调用processPacket()或loopPacket()方法时，还可以指定在该方法返回之前要捕获的数据包数。您可以指定-1继续无限量地捕获数据包。

回调的两个方法processPacket()和loopPacket()非常相似。通常您可能希望使用processPacket()，因为它支持超时和非阻塞模式，而loopPacket()不支持。
