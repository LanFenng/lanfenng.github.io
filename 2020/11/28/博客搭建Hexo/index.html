博客搭建Hexo

Posted on 2020-11-28

使用Hexo框架搭建博客

1.下载安装nodejs,官网：nodejs.org

找到LTS版本的For windows(x64)下载

这里下载的是node-v14.15.1-x64.msi

点击运行安装，根据习惯更改安装位置

2.运行node

用管理员模式运行命令提示符

输入node -v检查node版本、输入npm -v检查包管理器版本

3.利用npm安装一个cnpm,设置镜像源。

-g（全局安装） –registry（镜像源）=https://registry.npm.taobao.org

等待安装结束

4.使用cnpm安装Hexo博客框架

使用cnpm全局安装hexo博客框架cnpm install -g hexo-cli

使用hexo -v验证安装成功

5.检查当前路径，在D盘下建立blog文件夹

6.进入blog目录下，初始化Hexo

安装错误：BLOG文件夹必须为空

安装卡在这里：

CTRL+C尝试退出后：

191 packages和其他人几千的package相差很多
查看git版本：

卸载Git 重新安装Git,默认安装

清空BLOG文件夹里的内容，重新初始化Hexo
依旧卡在之前的地方，CTRL+C终止操作

重新查看镜像源发现镜像源没有改过来，更改镜像源npm config set registry https://registry.npm.taobao.org

查看更改镜像源：

清空BLOG文件夹，重新初始化Hexo
依然卡在第一行：

参考网上方法，设置Git代理

没有什么用，科学上网后，卡了一下，又很快安装了

重装系统后，重新部署，没有开代理，很顺利完成初始化

总结：之前应该是安装正确了，然后觉得不对，先继续配置，看看有没有问题

7.启动Hexo: hexo s

在浏览器里打开 http://localhost:4000，就可以看到默认创建的一篇博文

打开博客后这里会新增数据：

CTRL+C停止当前操作：

8.新建第一篇博文

使用命令：hexo n

查看目录，进入目录后，查看目录列表信息

打开并编辑，使用VS code，直接打开所在文件进行编辑保存

9.清理后，重新启动hexo，然后会看到新添加的文章

清理一下 hexo clean

生成一下 hexo g

重新启动一下 hexo s

浏览器打开 http://localhost:4000 新增内容

10.部署到Github上去

在Github上新建仓库，注意仓库名称必须是：”your Github name”.github.io

安装Git插件
cnpm install --save hexo-deployer-git

！敲错单词！>>>deplpyer

重新安装：

设置_config.yml

windows下用notepad打开

更改图示内容

修改后：

CTRL+S保存，然后关闭记事本

部署到远端 hexo d

报错：在108行有缩进错误，修改后：

产生错误：

网上查找原因：
大多是因为git进行push或者hexo d的时候改变了一些.deploy_git文件下的内容

解决办法
删除.deploy_git文件夹

输入git config --global core.autocrlf false

配置邮箱和名字

然后，依次执行：

点击后跳转到网页

点击授权，刷新链接，就可以看见博客已经部署上去了

11.主题更改

在GitHub上搜索hexo theme
找到这篇主题：https://github.com/iissnan/hexo-theme-next themes/next

下载主题

下载速度超级慢，大概16M的东西，差不多都是15k/s的速度，中间还经历了一次断网，总之很艰难得下载完成了

重装系统后，默认网络环境下，下载速度很快（晚上3：57）

启用主题
与所有 Hexo 主题启用的模式一样。 当 克隆/下载 完成后，打开 站点配置文件_config.yml， 找到 theme 字段，并将其值更改为 next
